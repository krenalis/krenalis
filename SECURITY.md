# Security Policy

## Reporting a vulnerability

Security vulnerabilities should not be reported publicly. Contact [security@krenalis.com](mailto:security@krenalis.com).

If you discover a potential security vulnerability, please follow this process:

1. **Do not disclose it publicly.**  
   Do not create public GitHub issues, discussions, or pull requests describing the vulnerability.

2. **Contact the security team directly.**  
   Send a detailed report to **security@krenalis.com** including:
    - A clear description of the issue
    - Steps to reproduce the vulnerability
    - A minimal proof of concept (if possible)
    - Affected versions and environment details

3. **Acknowledgment.**  
   The security team will acknowledge receipt within 5 days.

4. **Triage and response.**  
   The issue will be investigated, severity assigned, and next steps communicated within 5 business days.

5. **Coordinated disclosure.**  
   If confirmed, we will work with you to fix the issue before public disclosure.
   Your issue will be fixed or made public within 60 days.
   A security advisory will be published once patches are available and users have been notified.

6. **Credit.**  
   Researchers who follow this policy may be credited in the security advisory, unless they prefer to remain anonymous.
